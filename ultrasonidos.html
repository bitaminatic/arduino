Pasos de la práctica
1
¿Cómo funciona el HC-SR04?
Entender antes de conectar
📡

El sensor ultrasónico funciona como el sonar de un murciélago: emite un pulso de sonido y mide cuánto tarda en volver. Con ese tiempo, Arduino calcula la distancia.

Principio de funcionamiento
1
El pin TRIG (trigger) es la salida: Arduino envía un pulso de 10 microsegundos para activar el sensor.
2
El pin ECHO es la entrada: se queda en HIGH mientras el sonido viaja y vuelve. pulseIn() mide ese tiempo en microsegundos.
3
La fórmula para convertir el tiempo en centímetros es: distancia = tiempo ÷ 58 (el sonido va y vuelve a ~340 m/s).
⚠️

Rango del sensor: El HC-SR04 mide de 2 cm a 400 cm. Por debajo de 2 cm puede dar lecturas erróneas. Tenlo en cuenta al probar.

Conectar el circuito
HC-SR04 + LED + Buzzer en la protoboard
🔌

Antes de conectar nada: desconecta el cable USB de Arduino. Siempre monta el circuito sin corriente para evitar cortocircuitos.

Esquema de conexiones
Componente Pin del componente → Arduino
HC-SR04 VCC→ 5V
GND→ GND
TRIG→ Pin 9
ECHO→ Pin 10
LED rojo Ánodo (+, pata larga) → resistencia 220Ω → pin
Cátodo (−, pata corta) → GND		 → Pin 12
Buzzer + (pin positivo) → Pin 11
⚠️

No olvides la resistencia del LED. Sin ella, el LED puede quemarse y dañar el pin de Arduino. La resistencia de 220Ω es obligatoria.

3
Código parte 1: leer la distancia
Primero solo el sensor, sin actuar sobre nada
💡

Estrategia: Vamos por partes. Primero comprobamos que el sensor funciona y que leemos bien la distancia. Después añadimos el LED y el buzzer.

// ─── Práctica HC-SR04 — Paso 1: leer distancia ───────────────────

// 1. Definimos los pines del sensor
const int TRIG = 9;   // Pin de disparo (salida)
const int ECHO = 10;  // Pin de eco      (entrada)

void setup() {
  pinMode(TRIG, OUTPUT);  // TRIG es una salida
  pinMode(ECHO, INPUT);   // ECHO es una entrada
  Serial.begin(9600);     // Abrimos el Monitor Serie
}

void loop() {

  // 2. Enviamos el pulso de disparo
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);            // Limpiamos el pin
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);           // Pulso de 10 microsegundos
  digitalWrite(TRIG, LOW);

  // 3. Medimos el tiempo que tarda en volver el eco (en microsegundos)
  long duracion = pulseIn(ECHO, HIGH);

  // 4. Convertimos el tiempo en distancia (cm)
  long distancia = duracion / 58;

  // 5. Mostramos la distancia en el Monitor Serie
  Serial.print("Distancia: ");
  Serial.print(distancia);
  Serial.println(" cm");

  delay(200);  // Esperamos 200 ms antes de la siguiente medida
}
1
Copia el código en el IDE de Arduino y súbelo a la placa.
2
Abre el Monitor Serie (botón lupa arriba a la derecha, o Ctrl+Mayús+M). Asegúrate de que la velocidad está en 9600 baudios.
3
Acerca y aleja la mano del sensor. Deberías ver cómo cambia el valor de distancia en el monitor.
✅

¿Funciona? Si ves valores de distancia que cambian al mover la mano, ¡el sensor está funcionando! Si siempre da 0 o valores muy grandes, revisa las conexiones TRIG/ECHO.

4
Código parte 2: añadir LED y buzzer
Actuar según la distancia medida
🎯

Lógica del programa: Vamos a crear tres zonas de distancia. Según en qué zona esté el objeto, el LED y el buzzer se comportarán de forma diferente.

Zonas de distancia
// ─── Práctica HC-SR04 — Código completo ──────────────────────────

// Pines
const int TRIG   = 9;
const int ECHO   = 10;
const int LED    = 12;
const int BUZZER = 11;

void setup() {
  pinMode(TRIG,   OUTPUT);
  pinMode(ECHO,   INPUT);
  pinMode(LED,    OUTPUT);
  pinMode(BUZZER, OUTPUT);
  Serial.begin(9600);
}

void loop() {

  // ── 1. Leer el sensor ──────────────────────────────────
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);

  long duracion  = pulseIn(ECHO, HIGH);
  long distancia = duracion / 58;

  Serial.print("Distancia: ");
  Serial.print(distancia);
  Serial.println(" cm");

  // ── 2. Actuar según la distancia ──────────────────────
  if (distancia > 30) {
    // 🟢 ZONA SEGURA: todo apagado
    digitalWrite(LED,    LOW);
    digitalWrite(BUZZER, LOW);

  } else if (distancia > 10) {
    // 🟡 ZONA ATENCIÓN: LED encendido, 1 pitido corto
    digitalWrite(LED,    HIGH);
    digitalWrite(BUZZER, HIGH);
    delay(50);
    digitalWrite(BUZZER, LOW);

  } else {
    // 🔴 ZONA PELIGRO: LED parpadeante + alarma continua
    digitalWrite(LED,    HIGH);
    digitalWrite(BUZZER, HIGH);
    delay(100);
    digitalWrite(LED,    LOW);
    digitalWrite(BUZZER, LOW);
    delay(100);
  }

  delay(100);
}
1
Sube este código a Arduino. El comportamiento debería cambiar al acercar y alejar la mano del sensor.
2
Observa en el Monitor Serie los valores de distancia mientras pruebas las tres zonas.
3
Si el buzzer suena sin parar, revisa que el pin + del buzzer está en el pin 11 y el − en GND.
5
Verificación y prueba
Comprueba que todo funciona correctamente

Marca cada punto una vez que lo hayas comprobado:

El Monitor Serie muestra valores de distancia que cambian al acercar/alejar la mano.
Con la mano a más de 30 cm: LED apagado y buzzer en silencio.
Con la mano entre 10 y 30 cm: LED encendido fijo y buzzer da un pitido corto.
Con la mano a menos de 10 cm: LED parpadea y buzzer suena de forma intermitente.
Los valores de distancia son coherentes (no salen valores de 0 o 400 cm sin motivo).
🎉

¡Práctica completada! Todos los puntos verificados. Pasa al paso de ampliación para seguir practicando.

🔍

¿Algo no funciona? Repasa este listado:

  • El LED no enciende → revisa la resistencia de 220Ω y la polaridad del LED (pata larga = +).
  • El buzzer no suena → comprueba que el + del buzzer va al Pin 11 (no al GND).
  • La distancia siempre da 0 → probablemente ECHO y TRIG están intercambiados.
  • La distancia siempre da 400+ cm → asegúrate de que VCC del sensor va a 5V, no a 3.3V.

Para quienes quieran ir más lejos

⭐ Reto 1 — Múltiples LEDs

Añade un LED verde y uno amarillo. El verde se enciende en zona segura, el amarillo en zona atención y el rojo en zona peligro. Necesitarás dos pines más y dos resistencias.

⭐⭐ Reto 2 — Velocidad de pitido variable

En zona amarilla, usa delay() para que el buzzer pite más deprisa cuanto más cerca esté el objeto. Pista: calcula el delay en función de la distancia con map().

⭐⭐ Reto 3 — Mostrar en LCD

Si tienes una pantalla LCD I2C disponible, muestra la distancia en centímetros y el estado (SEGURO / ATENCIÓN / PELIGRO) en la pantalla en lugar del Monitor Serie.

⭐⭐⭐ Reto 4 — Aparcamiento inteligente

Simula un sensor de aparcamiento: 5 LEDs en línea que se van apagando a medida que el coche se acerca. El último LED y el buzzer se activan cuando la distancia es inferior a 5 cm.
